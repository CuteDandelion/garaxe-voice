# Decisions

Status: Living log
Last updated: 2026-07-14

| ID | Decision | Rationale |
|---|---|---|
| D-001 | Build customer-language intelligence, not generic sentiment analytics. | Interpretive, evidence-linked insight is the differentiator. |
| D-002 | Start with Google Business Profile plus flexible uploads. | Highest MVP value for local/multi-location businesses with low data-acquisition dependency. |
| D-003 | Use a modular monolith plus background worker. | Delivers reliable async processing without premature distributed complexity. |
| D-004 | Use a relational core, raw payload retention, and JSONB flexible edges. | Preserves unknown provider fields without sacrificing queryability. |
| D-005 | Make evidence traceability a hard invariant. | Prevents unsupported AI conclusions and builds trust. |
| D-006 | Use a hybrid deterministic/local-model/LLM pipeline. | Controls cost and improves reproducibility while retaining interpretation quality. |
| D-007 | Default UI to editorial Read mode with optional Investigate mode. | Matches Garaxe while preserving analytical depth. |
| D-008 | Defer full Copy Lab, alerts, CRM churn, and restricted provider connectors. | Protects the core validation scope. |
| D-009 | Reports are immutable snapshots; curation is separate from machine output. | Preserves reproducibility and user work. |
| D-010 | Use bounded, cost-aware agent delegation rather than automatic maximum fan-out. | Preserves parallel speedups while limiting duplicated context, token use, merge conflicts, and repeated verification. |
| D-011 | Route orchestration by economy/standard/deep capability tiers and escalate only on evidence. | Keeps model choices portable and directs expensive reasoning toward high-risk ambiguity instead of routine work. |
| D-012 | Establish the MVP with a React/Vite editorial Overview vertical slice before provider integration. Superseded for current navigation by D-039. | Locked the initial customer-facing data contract and visual system while OAuth/ingestion proceeded behind a stable UI boundary. |
| D-013 | Prove CSV ingestion client-side before adding persistence and asynchronous import jobs. | Validates mapping and quality UX cheaply while keeping server-authoritative parsing, security, and tenancy as explicit backend work. |
| D-014 | Use PGlite as the local PostgreSQL-compatible MVP adapter, not as the production database. | Provides real durable SQL and fast isolated integration tests without cloud credentials while preserving a migration path to managed PostgreSQL/Supabase. |
| D-015 | Freeze analysis configuration and persist every review membership before interpretive processing. | Makes inclusion/exclusion auditable, prevents moving datasets, and lets later signals/themes remain reproducible. |
| D-016 | Keep preprocessing deterministic and model-free. | Controls cost, preserves original language, and makes early pipeline failures independently testable before LLM-backed extraction. |
| D-017 | Ship the initial evidence and synthesis pipeline without an LLM completion dependency. Superseded for current default presentation by D-050; retained as the deterministic degraded fallback. | Made the early MVP usable without credentials or variable inference costs while preserving the later governed enrichment boundary. |
| D-018 | Persist spans, themes, and relationships instead of generated prose alone. | Allows every conclusion to be traversed to exact source language and re-evaluated when taxonomy versions change. |
| D-019 | Treat low-volume projects as exploratory evidence, not high confidence. | Small runs may validate a theme for product demonstration while confidence labels and contradiction metrics clearly expose weak support. |
| D-020 | Model curation as an append-only event stream and publish only a ready projection. | Preserves machine artifacts and auditability while ensuring rejected claims cannot leak into the authoritative Voice Map. |
| D-021 | Render PDFs from the immutable report snapshot through a replaceable local Python adapter. | Keeps the MVP report reproducible and visually testable without coupling publication to live database state; production can move the same snapshot contract to a managed worker. |
| D-022 | Establish tenant isolation with hashed opaque sessions and an organization compatibility bridge before selecting a production identity provider. | Secures every existing resource now while keeping password/social login replaceable and avoiding a destructive local migration. |
| D-023 | Keep one database interface across PGlite and node-postgres. | Preserves fast local integration tests while making managed PostgreSQL selection an environment/deployment concern rather than an application rewrite. |
| D-024 | Mark Google incremental sync and write capabilities false until live proof. | Prevents OAuth success or documented endpoints from being misrepresented as production-ready access. |
| D-025 | Preserve original upload artifacts separately from normalized mapping CSV. | XLSX/JSON conversion must not destroy immutable source provenance or prevent later reprocessing. |
| D-026 | Freeze selected Google locations into each sync job. | Prevents an in-flight import from changing when a user edits connection selection and keeps provider acquisition reproducible. |
| D-027 | Provide owner-email session recovery only on non-production loopback. | Keeps the local MVP usable after restart without pretending to be a production identity provider or exposing passwordless login publicly. |
| D-028 | Force every automated test database to `memory://`. | Prevents integration tests from truncating or contaminating developer/customer-like local data. |
| D-029 | Force database RLS and inject the authenticated user only through transaction-local configuration. | Makes tenant isolation a database backstop without leaking identity across pooled connections; the application authorization layer remains independently enforced. |
| D-030 | Approve OpenCode Go as an enrichment adapter behind a durable quota-aware queue. D-049 and D-050 later made interpretation exhaustive and the default completion gate when configured. | Added interpretation quality without making provider availability or credentials an evidence dependency; deterministic fallback remains explicit. |
| D-031 | When verified-price spend enforcement is enabled, enforce global, organization, project, and analysis-run budgets through worst-case integer-micro reservations and atomic reconciliation. D-049 made monetary enforcement opt-in. | Prevents overspend under concurrency and avoids floating-point accounting drift while preserving an auditable ledger for providers with verified pricing. |
| D-032 | Apply per-provider/model request and token buckets, bounded tenant fairness, leases, retries, dead-lettering, and circuit breakers. | Converts external quota and outage behavior into controlled asynchronous states instead of blocking HTTP requests or analysis runs. |
| D-033 | Keep deterministic artifacts authoritative and accept model output only as schema-valid, tenant-valid, exact-span evidence candidates. | Preserves zero-LLM publication, evidence traceability, and reproducibility while containing prompt injection and unsupported claims. |
| D-034 | Do not promote any currently evaluated OpenCode Go model to default enrichment. | MiniMax failed taxonomy/span validation, DeepSeek missed bounded completion, and Qwen timed out; deterministic analysis remains the only default until the published quality/latency gates pass. |
| D-035 | Enforce global, provider/model, and organization concurrency at atomic lease admission rather than in worker memory. | Transactional policy locks plus unexpired active-lease counts prevent multi-worker oversubscription, preserve tenant fairness, and ensure capped work consumes no request/token quota. |
| D-036 | Supplement governed signal rules with deterministic dataset-frequency aspect discovery. Superseded as the production theme source by D-037. | Preserved exact evidence while proving dataset-derived language before semantic clustering replaced rule/frequency production themes. |
| D-037 | Replace production keyword/frequency extraction with exact segmentation, pinned multilingual-e5-small ONNX q8 embeddings, deterministic clustering, and dataset-derived cluster representation. | Removes predetermined theme vocabulary while retaining CPU deployment, reproducibility, and exact evidence spans. |
| D-038 | Defer fine-grained signal typing to a SetFit classifier trained from real analyst curation. | Prevents invented labels from becoming product truth and creates an auditable quality-improvement loop. |
| D-039 | Make Voice Map Read the project landing view and remove Overview. | Eliminates duplicate product surfaces while preserving the accepted editorial hierarchy. |
| D-040 | Show full source feedback with exact-span highlighting in every Voice Map evidence item. | A short excerpt such as a single adjective cannot establish context, negation, or evidentiary trust. |
| D-041 | Use an accessible SVG evidence-bucket field with bounded support sizing and no WebGL. | Adds useful comparison and evidence interaction within the 4 vCPU/8 GB, reduced-motion, keyboard, and 390px constraints. |
| D-042 | Label bubbles with the evidence bucket/theme name and size them by independent supporting reviews; keep full customer feedback in the linked evidence drawer. | A sample sentence is not the value being compared and is often too long to remain legible inside a bubble. |
| D-043 | Derive bucket names from supported two-to-four-word phrases counted once per review; scale label type with bubble size and keep motion clearly perceptible behind collision resolution. | Single context tokens such as “during” are ambiguous, repeated segments can distort frequency, and uniform small type hides the visual hierarchy. |
| D-044 | Separate authorized-project selection from project creation and derive signed-in identity from the authenticated session. | A create-only control cannot switch existing workspaces, while hardcoded account labels obscure which identity owns the active session; explicit server-backed logout completes the boundary. |
| D-045 | Make the global date window create a new immutable analysis run. | A visual-only date button is misleading, while mutating a published artifact would violate reproducibility; a new run makes the selected evidence window real and auditable. |
| D-046 | Classify sentiment per exact clause before polarity-specific clustering and prefer recurrent cause language in representation. | Review-level ratings blur mixed feedback, and naming consequences does not tell an operator what to fix; clause polarity and root-cause phrasing preserve both the good and bad while keeping evidence exact. |
| D-047 | Add a feature-flagged, root-cause-first OpenCode Go cluster-interpretation job after deterministic theme persistence. | A bounded model can produce sharper aspect and praise/pain labels without becoming the evidence source or blocking the zero-LLM analysis path. |
| D-048 | Benchmark a sub-500 MB local GGUF interpreter as a provider-compatible fallback, but promote no model by size alone. | Local execution can improve privacy and cost resilience, while identical evidence, quality, and latency gates prevent a weak compact model from silently degrading interpretation. |
| D-049 | Make OpenCode interpretation exhaustive across validated themes and govern it by request/token capacity and concurrency; monetary budgets are opt-in only with verified pricing. | Capacity controls prevent rate-limit bursts without an invented cumulative spend ceiling silently degrading dashboard coverage. |
| D-050 | Make validated LLM cluster interpretation the default completion gate before curation, with deterministic output labeled only as degraded fallback. | Human review is not meaningful when raw frequency labels appear before model interpretation; the run must expose coverage and failure state instead of presenting fallback output as equivalent intelligence. |
| D-051 | Derive live post-click LLM progress from persisted queue and theme state, and reattach the Analysis workspace to an active run after navigation or reload. | Long provider-backed work must remain observable and recoverable without treating browser timing as truth; separate job completion from interpretation coverage so fallback is visible. |
| D-052 | Run Qwen cluster interpretation without thinking and constrain the versioned prompt to compact four-theme JSON under 1,200 tokens while retaining the independent 1,800-token provider cap. | The cap alone truncated 14/18 verbose responses; the compact pilot accepted 12/12 candidates with normal stops and materially lower output volume. This supersedes D-034 for local default inference while paid-beta promotion gates remain open. |
| D-053 | Set the local global, provider/model, and organization LLM concurrency ceilings to two. | The serial E2E monitor completed only 3/18 jobs in 92 seconds, while two simultaneous compact production-schema calls accepted 7/7 candidates and both stopped normally within 27.5 seconds. |
| D-054 | Replace forced centroid assignment with versioned mutual-KNN communities, deterministic coherence gates, and honest outliers; reuse the existing interpretation call for ambiguous `keep`/`split` advice. | Forced membership and per-segment labels created internally inconsistent themes. Reciprocal similarity plus coherence gates improves membership without extra LLM calls, while human review remains authoritative for any split. This supersedes D-019 only for singleton machine-theme promotion: a one-claim run remains inspectable but cannot manufacture a semantic cluster. |
| D-055 | Add an LLM publication disposition to cluster interpretation and preserve multi-label signal types through presentation. | Semantic proximity alone can reward shared templates and context. An auditable discard gate suppresses non-feedback clusters without deleting evidence, while multi-label projection prevents valid objections and emotions from disappearing behind one primary type. |
| D-056 | Calibrate mutual-KNN to the pattern-diverse gold set and route lower-cohesion accepted clusters through the existing LLM grouping decision. | A conservative 0.90 floor discarded useful paraphrases, while lower floors overmerged unrelated issues. The 0.84/0.81 hybrid improves coverage without adding calls, and unresolved split recommendations cannot publish. |
| D-057 | Publish report snapshot v2 with a compact evidence-cited LLM brief, deterministic charts, actions, and full customer comments. | Pattern labels alone do not support business decisions. Theme-ID validation and explicit fallback provenance keep guidance useful without weakening immutable evidence or making provider availability a publication dependency. |
| D-058 | Deploy the first public demo on the existing Bluerose Kubernetes cluster behind its Cloudflare Tunnel, using immutable GHCR images and retained single-node PostgreSQL. | The current app needs a persistent Node process and CPU-heavy ONNX runtime; the existing cluster has the required capacity and avoids misrepresenting a serverless host as a fit. The topology is explicitly staging, not HA or paid-beta production. |
| D-059 | Add an explicit access-key session route only for the staging deployment tier. | Production disables loopback recovery and no identity provider is selected yet; a generated high-entropy staging secret keeps the demo recoverable without exposing email-only login or weakening the production IdP gate. |
| D-060 | Enable the evaluated OpenCode Go interpreter on Bluerose with explicit capacity policy and a public-HTTPS staging egress exception. | The public demo should exercise its intended interpretation layer. Standard NetworkPolicy cannot select the provider FQDN, so private/local/reserved destinations remain denied, the adapter owns the fixed URL, Google remains unconfigured, and production still requires domain-aware egress plus managed secret rotation. |
